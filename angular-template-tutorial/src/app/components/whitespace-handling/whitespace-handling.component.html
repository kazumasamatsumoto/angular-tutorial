テンプレート内の空白

デフォルトで、Angular テンプレートはフレームワークが不要とみなす空白を保持しません。 これにより、生成されるHTMLのサイズが小さくなり、パフォーマンスが向上します。 ただし、場合によっては空白を保持したい場合があります。

デフォルトの空白処理

デフォルトでは、Angular は複数の連続した空白を1つの空白に圧縮します。 また、要素間の空白も削除されます。

コード例

<p>Hello         world</p>

レンダリング結果

Hello world

※ 複数の空白が1つに圧縮されています

空白を保持する

コンポーネントレベルで空白を保持するには、preserveWhitespacesオプションを使用します。

コンポーネント定義での設定

@Component({
  /* ... */,
  preserveWhitespaces: true,
  template: `
    <p>Hello         world</p>
  `
})

レンダリング結果（空白保持）

Hello world

※ このコンポーネントではpreserveWhitespaces: trueが設定されているため、複数の空白が保持されています

特別なHTML空白エンティティ

Angular には、空白を制御するための特別なHTML空白エンティティ&ngsp;があります。 このエンティティは、コンパイルされた出力で保持される単一のスペース文字を生成します。

コード例

<p>Hello&ngsp;&ngsp;&ngsp;world</p>

レンダリング結果

Hello&ngsp;&ngsp;&ngsp;world

※ &ngsp;エンティティによる空白が保持されています

空白処理のベストプラクティス

  • デフォルトの空白処理は、ほとんどの場合で問題ありません。
  • 特定のレイアウトで空白が重要な場合は、preserveWhitespaces: trueを使用するか、&ngsp;エンティティを使用します。
  • CSS の white-space プロパティを使用して、テキストの空白処理を制御することもできます。
  • パフォーマンスを最適化するには、グローバルに空白を保持するのではなく、必要なコンポーネントでのみ空白を保持するようにしましょう。
